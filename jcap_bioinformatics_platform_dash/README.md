# JCAP Bioinformatics Platform Dash Frontend

A Dash frontend for the modular JCAP bioinformatics platform. It is designed to call three separate API modules:

```text
8000 = RNA-seq API
8001 = Enrichment Network API
8002 = Enrichment Triage / LLM API
8050 = Dash frontend
```

## Current layout

The app has a hero header and three major tabs:

1. **Omics**
   - RNA-seq subtab
   - Upload counts and phenotype metadata
   - Submit to the RNA-seq API
   - Show job/status JSON and downloaded output bundle files when available

2. **Network Visualization**
   - Upload enrichment edge files, usually `gene,term,adjusted_pvalue`
   - Build graph through the Network API
   - Render bipartite graph and pathway projection graph
   - Show candidate, diffusion, consensus, node, and edge tables
   - Download graphs as SVG and tables as CSV

3. **LLM Triage**
   - Upload enrichment file
   - Enter phenotype/context fields
   - Run pretriage or full PubMed/LLM mode through the Triage API
   - Display Markdown report, PDF, programs, triage rows, claims, and raw JSON
   - Download bundle, Markdown, and PDF

```bash

```

## Notes

The Network and Triage panels are fully wired to the endpoints we validated.

The RNA-seq panel submits jobs to `/rnaseq/jobs` and tries common status/download endpoints. If your RNA API response includes `status_url` and `download_url`, the frontend will use those directly. If the RNA API currently returns only `job_id`, the panel will still show the job response, but we may want to add a formal `/rnaseq/jobs/{job_id}` status endpoint and `/rnaseq/jobs/{job_id}/download` bundle endpoint to make output display completely automatic.

## v2 RNA output behavior

The RNA-seq panel now has a **Fetch/extract RNA bundle** button. After submitting the job and polling until it finishes, click that button. The app will try common bundle endpoints such as:

```text
/rnaseq/jobs/{job_id}/download
/rnaseq/jobs/{job_id}/bundle
/rnaseq/jobs/{job_id}/results.zip
```

If the RNA API response includes `download_url`, `bundle_url`, `result_url`, or `results_url`, the frontend will use that URL first.

Once the bundle is fetched, the UI displays the actual module outputs:

```text
DE results
Full limma table
Enrichment tables
Classifier tables
Power analysis tables
Plot HTML outputs
Output file list
```

RNA plots generated by the current R CLI are HTML widgets. The frontend displays them inline and lets you download the plot HTML. For true SVG downloads from RNA plots, the RNA CLI/API should also emit static SVG files or return Plotly JSON. The Network API graphs already support SVG export because they are generated from Plotly figure JSON in the frontend.
